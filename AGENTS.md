# AGENTS.md

This file provides guidance for AI coding agents working on this codebase.

## Before Making Plans or Changes

**Always consult the following documents before proposing or implementing changes:**

### For Code Architecture & Engineering Decisions
→ Read **`CONTRIBUTING.md`**

This covers:
- Feature-first architecture and file organization
- Page component guidelines ("skinny page" pattern)
- Logic isolation and where to put business rules
- Naming conventions and code organization
- Pull request requirements and checks

### For UI/Design & Styling Decisions
→ Read **`Design Rules.md`**

This covers:
- Theme system and color palette
- How to use and change the design system
- Button variants and when to use each
- Typography and font usage
- Visual hierarchy principles

## Summary

| Type of Change | Consult |
|----------------|---------|
| Adding/modifying features | `CONTRIBUTING.md` |
| Refactoring or moving code | `CONTRIBUTING.md` |
| UI components or styling | `Design Rules.md` |
| Colors, buttons, theme | `Design Rules.md` |
| Both architecture + UI | Both documents |
